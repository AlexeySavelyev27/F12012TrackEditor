# Карта файловых форматов трассы Melbourne (F1 2012)

В каталоге `melbourne/` находятся файлы различных типов. Ниже приведена таблица, описывающая основные форматы и их заголовки.

| Расширение и пример | Заголовок (первые 4 байта) | Назначение/формат |
|--------------------|---------------------------|------------------|
| `*.xml` – например `RacingLine.xml` | `<?xml` или BOM `EF BB BF` | Обычный XML‑текст. |
| `*.xml` с заголовком `00 42 58 4D` – например `basewind2.xml` | `00 42 58 4D` | BXML‑формат Codemasters (бинарный XML). |
| `*.xml` с заголовком `1A 22 52 72` – например `ambience_triggers.xml` | `1A 22 52 72` | Другой вариант бинарного XML. |
| `*.pssg` (и `*.ens`) – например `objects.pssg` | `50 53 53 47` ("PSSG") | Архивы EGO Engine; внутри могут быть модели и текстуры. |
| `*.jpk` – например `route_0/track.jpk` | `4A 50 41 4B` ("JPAK") | Контейнеры данных трассы. |
| `*.clm` – например `route.clm` | `8E C2 39 C4` | Коллизии/геометрия (формат EGO). |
| `*.cqtc` – например `boundarylines.cqtc` | `73 1A 45 C4` | Линии границ и ресеты трассы. |
| `*.bin` – например `trackinfo.bin` | `08 7F 01 7C` | Различные бинарные таблицы (деревья, объекты, инфо о трассе). |
| `*.dat` – `tod.dat` | `54 4F 44 44` ("TODD") | Временные таблицы (Time Of Day). |
| `*.grs` – `grass.grs` | `04 00 00 00` | Параметры травы. |
| `*.lut` – `trackinfo.lut` | `C4 8A C4 8A` | Таблицы поиска (Lookup Table). |
| `*.vis` – `track.vis` | `04 00 00 00` | Данные видимости. |
| `route_0/Reflections.xml` | Начинается с `<Reflections>` | Пример обычного XML внутри подкаталога. |

Всего в каталоге `melbourne/` насчитывается 61 файл. По расширениям: 29 XML (часть из них бинарные), 17 PSSG/ENS, по нескольку BIN, CLM, CQTC и других типов.
